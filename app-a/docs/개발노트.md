## 1. 파일 구조 (index.html 내부)

전부 한 파일. 순서:

1. `<head><style>` — 전 CSS (엑셀 룩앤필 + 차트/위저드/로딩 스타일)
2. `<body>` HTML 골격 — 타이틀바, 리본, 수식줄, 그리드 테이블, 차트 뷰 컨테이너, 시트탭, 상태바, 오버레이들
3. `<script>` — 전 로직. 대략의 섹션 순서:
   - 기본 상태/상수 (`COLS=40, ROWS=300`, `sheets`, `sel`, `aR/aC` …)
   - 그리드 생성 `buildGrid()`
   - **수식 엔진** (렉서 → 파서 → 평가기 → 40여 개 함수)
   - 표시 형식 `fmtNum` · 셀 렌더 `renderCell` · 재계산 `recalc`
   - 선택/마우스/키보드/편집/클립보드/채우기 핸들
   - 리본 동작(서식·정렬·팔레트) · 탭 전환 · 더미 패널 생성
   - 시트 탭 · 확대축소 · 컨텍스트메뉴 · 백스테이지 · 토스트
   - **측정 QC 시트 빌더** `buildQCSheet` / 더미 생성 `genQCData`
   - **템플릿/업로드/CSV 파싱/데이터도구** (텍스트나누기·중복제거)
   - **차트 시스템** (추출 `extractQC` → 통계 → SVG 생성 → `renderCharts`)
   - **차트 내보내기** (SVG→PNG 래스터화, 복사/저장, 몽타주)
   - **튜토리얼 위저드**
   - 초기화

> 편집 팁: 함수는 대부분 `function` 선언(호이스팅)이라 정의 순서에 관대하지만, `let`/`const` 전역(`qcSource`, `chartToken`, `WZ_STEPS` 등)은 실행 시점(초기화 이전)에 초기화되어야 한다. 이벤트 핸들러 내부에서만 참조되면 안전.

## 2. 데이터 모델

### 2.1 통합 문서 / 시트
```js
sheets = [ sheet0, sheet1, ... ]   // 배열, cur = 활성 인덱스
sheet() = sheets[cur]
```
시트 두 종류:
```js
// 그리드 시트
{ name, data:{}, colW:{}, rowH:{} }               // kind 없음 = 'grid'
// 차트 시트
{ name, kind:'chart', chart:'normal'|'scatter'|'box', data:{}, colW:{}, rowH:{} }
```

### 2.2 셀 (희소 저장)
`data`는 `"r,c" → cell` 딕셔너리. 빈 셀은 키 자체가 없음(희소). 40×300=12,000칸이지만 채운 칸만 저장.
```js
cell = {
  v: "10.153",          // 원본 문자열. "=SUM(A1:A3)"이면 수식, 앞이 "'" 이면 강제 텍스트
  s: {                  // 선택적 스타일(플랫 객체)
    b, i, u, st,        // 굵게/기울임/밑줄/취소선
    ff, fs,             // 글꼴, 크기(pt)
    fill, col,          // 채우기색, 글자색
    al,                 // 'left'|'center'|'right'
    wrap,               // 자동 줄바꿈
    fmt,                // 'general'|'comma'|'currency'|'percent'
    dec                 // 소수 자릿수
  }
}
```
**중요한 함정**: 여러 셀에 같은 스타일 객체 레퍼런스를 공유하면 안 됨(한 셀 편집이 전체에 전파). QC 빌더의 `put`은 `Object.assign({}, s)`로 매 셀 독립 복제. (테스트로 공유 여부 검증함)

### 2.3 실행 취소(undo)
`record(chs,r,c)` 로 변경 전 스냅샷 수집 → 변경 → `finish(chs)` 가 변경 후 스냅샷을 찍고 실제로 바뀐 것만 스택에 push. `undoStack`/`redoStack` 각 100단계. 시트 간 이동 변경도 `sh` 필드로 추적.

## 3. 수식 엔진 (직접 구현, 라이브러리 없음)

파이프라인: **렉서(`lex`) → 재귀하강 파서/평가기(`evalFormula`) → 셀 평가(`getVal`)**

- **렉서**: 숫자/문자열("…", "" 이스케이프)/참조(A1, $A$1)/함수명/연산자/불린 토큰화. 한글 함수명 문자도 허용.
- **파서(우선순위)**: 비교(`= <> < > <= >=`) → 문자열결합(`&`) → 덧셈 → 곱셈 → 단항(±) → 거듭제곱(`^`) → 후위(`%`) → 기본식(숫자/문자열/참조/범위 `A1:B3`/함수/괄호).
- **범위**는 `{rng:[r1,c1,r2,c2], vis}` 객체로 전달, 함수가 `flatVals`로 펼침.
- **평가값 타입**: `number | string | boolean | '' | {err:'#DIV/0!'|…}`.
- **순환 참조** 감지: `getVal`이 `vis`(Set)에 방문 중인 셀 키를 넣고, 재진입 시 `#REF!`.
- **에러 전파**: `#DIV/0! #VALUE! #NAME? #REF! #N/A #NUM!`. `IFERROR`가 잡을 수 있도록 함수 인자는 `parseArg`가 try/catch로 `{err}`를 감싸 전달(값 인자만; 지연 평가 아님).

**구현 함수(40+)**: SUM AVERAGE COUNT COUNTA COUNTBLANK MAX MIN · IF IFERROR AND OR NOT · ROUND ROUNDUP ROUNDDOWN ABS INT MOD SQRT POWER PI RAND RANDBETWEEN · LEN LEFT RIGHT MID UPPER LOWER TRIM REPT CONCAT CONCATENATE TEXTJOIN · TODAY NOW · COUNTIF SUMIF AVERAGEIF · VLOOKUP.

`COUNTIF(range, ">"&셀)` 형태: `makeCrit`가 `">10.15"` 문자열을 연산자+숫자로 파싱. QC 요약이 이 방식으로 상·하한 초과 개수를 실제 수식으로 계산한다.

**재계산 전략**: `recalc()`는 캐시(`cache`)를 비우고, 직전에 채워졌던 칸(`prevFilled`) ∪ 현재 칸을 다시 렌더. 전체 시트가 아니라 "채운 칸"만 순회 → 희소 데이터에 효율적.

## 4. 그리드 UI (엑셀 재현)

- **DOM**: 실제 `<table>` (sticky `<th>` 헤더/행번호). 40열×300행 = 약 12,000 `<td>`. 셀 내용은 채운 칸만 textContent/스타일 지정.
- **선택**: `sel={r1,c1,r2,c2}` + 활성셀 `aR,aC`. 초록 테두리(`#sel-border`), 큰 범위는 개별 틴트 대신 반투명 오버레이. 헤더 하이라이트, "복사 점선(개미행렬)" CSS 애니메이션.
- **편집**: `#cell-editor` 오버레이 input. `quick`(바로 타이핑, IME 한글 유지) / `incell`(F2·더블클릭) 모드. 수식 입력줄과 양방향 동기화.
- **키보드**: 방향키/Enter/Tab 이동, Shift 확장, Ctrl+방향(끝으로 점프 `ctrlJump`), Ctrl+A/Z/Y/B/I/U/S, Delete, PageUp/Down.
- **클립보드**: 내부 복사 시 스냅샷 + TSV. 붙여넣기 시 상대 참조 자동 변환(`translateFormula`), 외부 TSV(진짜 엑셀 복사)도 파싱. 잘라내기 이동.
- **채우기 핸들**: 드래그로 복사 / 숫자 연속 채우기(등차).
- **열/행 리사이즈**: 헤더 경계 드래그 → `colW/rowH`.

## 5. 측정 QC 시트 (`측정데이터`)

레이아웃(1-indexed 행): 1 제목 · 2 포인트No · 3 스펙 · 4 상한(USL) · 5 하한(LSL) · 6 데이터헤더 · 7~206 시리얼 200개 · 이후 요약(검사수/평균/최소/최대/상한NG/하한NG/판정).

- **빌더 분리**: `buildQCSheet(name, pts, serials)` 가 셀·스타일·요약수식을 채움. `pts[c] = {no, spec, usl, lsl, values:[...]}`, `serials=[번호…]`.
  - 더미 `genQCData`: 정규분포 노이즈(Box–Muller)로 값 생성 + 소수의 강제 불량. 공칭 3~15, 공차 ±0.05~0.15.
  - CSV 업로드: `parseMeasurementCSV`가 같은 `{pts, serials}` 구조로 파싱 → 같은 빌더 재사용.
- **NG 색칠**: 값 > USL → 빨강, < LSL → 노랑. 요약의 상·하한 NG 셀/판정 셀도 색.
- **요약은 진짜 수식**: `=AVERAGE(B7:B206)`, `=COUNTIF(B7:B206,">"&B4)`, `=IF(B211+B212=0,"OK","NG")` … 값을 고치면 재계산됨.

## 6. 데이터 가져오기/내보내기 (실동작으로 만든 더미 아이콘)

| 리본 위치 | 동작 |
|---|---|
| 데이터 ▸ 측정데이터 ▸ **템플릿 받기** | 현재 양식 CSV(`측정데이터_템플릿.csv`) 생성·다운로드. UTF-8 BOM(엑셀 한글). |
| 데이터 ▸ 측정데이터 ▸ **업로드** | 파일 선택 → **xlsx(바이너리) 또는 CSV/TSV 자동 판별**(첫 2바이트 `PK`=xlsx) → `parseMeasurementRows` → `buildQCSheet`. 형식 미인식 시 더미로 폴백. |
| 데이터 ▸ 측정데이터 ▸ **측정 더미 불러오기** | 새 랜덤 데이터 생성 → 새 시트. 차트 소스(`qcSource`) 갱신. |
| 데이터 ▸ **텍스트/CSV** | 임의 CSV/TSV → 새 그리드 시트로 가져오기(구분자 자동 감지). |
| 데이터 ▸ **모두 새로 고침** | 그리드=재계산 / 차트=재렌더. |
| 데이터 ▸ **텍스트 나누기** | 선택 열을 구분자(prompt)로 분리해 옆 열에 채움. |
| 데이터 ▸ **중복된 항목 제거** | 선택 범위 행 중복 제거(첫 항목 유지, 위로 당김). |
| 데이터 ▸ 정렬(오름/내림) · 홈 ▸ 자동합계/정렬/지우기/서식 등 | 기존 실동작. |

파일 입력은 숨은 `#file-input` 하나를 `uploadMode('qc'|'csv')`로 분기. 실제 업로드는 async `processUpload(file, mode)`.

### xlsx 직접 읽기 (라이브러리 없이)
`.xlsx`는 XML을 담은 ZIP 아카이브. 외부 라이브러리 없이 3단계로 읽는다:
1. **ZIP 파싱** `unzipXlsx(buf)`: 끝의 EOCD(End Of Central Directory) 시그니처를 뒤에서 찾아 중앙 디렉터리 위치를 얻고, 각 엔트리(이름·압축방식·크기·로컬헤더오프셋)를 순회. 필요한 엔트리(`xl/sharedStrings.xml`, `xl/worksheets/sheetN.xml`)만 골라 처리.
2. **DEFLATE 해제** `inflateRaw(bytes)`: 브라우저 내장 **`DecompressionStream('deflate-raw')`** 사용(Chrome/Edge/Firefox/Safari 최신). 라이브러리 불필요.
3. **XML 파싱** `readXlsxRows`: sharedStrings(`<si><t>`)를 인덱스 배열로, 워크시트 셀(`<c r="A1" t="s"><v>idx</v></c>` / 숫자 / inlineStr)을 정규식으로 읽어 `rows[][]` 생성 → `parseMeasurementRows`로 전달.

구형 `.xls`(바이너리 BIFF)는 ZIP이 아니라 미지원 → 텍스트 경로로 빠져 파싱 실패 → 더미 폴백.

**여전히 더미(시각 장식)**: 조건부 서식·표 서식·피벗·차트 삽입(리본)·필터·유효성검사·병합·페이지레이아웃 등. 클릭 시 "더미" 토스트. 나중에 `dummy` 클래스를 떼고 핸들러만 붙이면 되는 구조.

## 7. 차트 시스템 (SVG 직접 생성)

- **추출** `extractQC()`: `qcSource` 시트에서 포인트별 값 배열 수집 + 통계 계산 — 평균, 표준편차(n-1), min/max, 사분위(Q1/중앙/Q3, 선형보간), 상·하한 NG 수, **Cp/Cpk**.
- **렌더** `renderCharts(type, cv)`: 요약 타일(종합판정·포인트수·시료수·총NG·NG포인트·최저Cpk) + 범례 + 포인트별 카드 30개. 박스플롯 시트는 상단에 규격 정규화(±1) 전체 비교 차트.
- **차트 3종**(문자열로 SVG 조립):
  - **정규분포** `svgNormal`: 히스토그램(11 bin) + 정규분포 곡선(PDF×N×binWidth). 값이 가로축.
  - **산점도** `svgScatter`: 시료 순서 추세선 + 점(이탈=빨강). 값이 세로축.
  - **박스플롯** `svgBox`: 사분위 상자·중앙값·평균(◆)·1.5 IQR 수염·이상치.
- **읽기 쉽게 한 규칙(디자인)**:
  - 규격선과 글자 겹침 방지: 상/하한 태그를 **플롯 바깥 여백**(정규분포=위, 산점도/박스=오른쪽)에 배치. 차트별 개별 마진.
  - 상한/평균/하한 **값은 하단 스트립**에 색 구분(상한 빨강 / 평균 주황 / 하한 파랑).
  - 정상 범위(상~하한) 초록 밴드, 이탈 구간 옅은 빨강 밴드.
  - 카드 통계는 별도 박스(평균·σ·Cpk·불량률), Cpk<1.33 빨강.
- **색 팔레트 근거**(dataviz 스킬): 데이터=파랑 `#2a78d6`, 이탈/상한=빨강 `#d03b3b`, 하한=파랑 `#2f6fd0`, 목표=초록 `#0ca30c`(status good), 평균/정규곡선=주황 `#eb6834`. 판정은 색만이 아니라 OK/NG 텍스트 배지 병행(색각 접근성).

### 내보내기 (복사/저장)
- **래스터화**: SVG 문자열 → `data:image/svg+xml;charset=utf-8,`+`encodeURIComponent`(한글 안전) → `<img>` → `<canvas>`(흰 배경) → PNG blob.
- **단일 카드**: 호버 시 우하단 `복사/저장`. `buildExportSVG`로 제목·스펙·통계 라벨까지 구운 독립 SVG를 만들어 내보냄.
- **다중 선택**: 카드 체크박스 → 상단 선택 바 → 선택분을 격자 몽타주 PNG 한 장으로.
- **전체**: 제목 오른쪽 끝 `전체 복사/저장` → 30개 몽타주.
- 복사(`navigator.clipboard.write` + `ClipboardItem`) 미지원 브라우저는 자동으로 파일 저장 폴백.

## 8. 튜토리얼 위저드 (5단계)

말풍선 + 스포트라이트(`box-shadow: 0 0 0 9999px` 로 주변 딤). `WZ_STEPS` 배열: 각 스텝은 `{title, body, place, before(done), target()}`.

1. 데이터 불러오기 (데이터 탭 하이라이트) — `before`가 `activateTab('data')`
2. 다음 시트 클릭 (시트 탭 하이라이트)
3. 위=요약, 아래=상세 (정규분포로 전환 후 타일 하이라이트) — `wzEnsureChart`가 시트 전환+로딩 대기
4. 카드 하나 복사·저장 (첫 카드 액션 강제 노출 `wz-reveal`)
5. 여러 개 선택·한 번에 저장 (카드 3개 자동 체크 → 선택 바 하이라이트)

- 최초 1회 자동 실행(`localStorage['qc_tut_done']`), 종료 후 좌하단 "다시 보기" 힌트. 데이터 탭 **분석 마법사** 버튼으로도 재생.
- `positionWz()`가 대상 `getBoundingClientRect`로 스포트라이트+말풍선+화살표 배치. 리사이즈 시 재배치.
- `wzCleanup()`이 스텝 전환/종료 때 데모 상태(강제 노출·선택) 정리.

## 10. 테스트 전략 (브라우저 없이 Node 검증)

브라우저 자동화가 없어, **인라인 `<script>`를 정규식으로 추출**하고 필요한 함수 블록만 슬라이스해 Node 모듈로 감싸 순수 로직을 검증했다.

- **수식 엔진**: 32개 케이스(사칙·비교·`&`·`%`·범위·SUMIF/COUNTIF/VLOOKUP·순환참조 `#REF!`·IFERROR) 기대값 비교.
- **QC 요약 수식**: 시트에 저장된 `=AVERAGE/COUNTIF/IF` 를 `getVal`로 평가해 원자료 재계산과 30열 전부 일치 확인.
- **차트 파이프라인**: `extractQC`(30/200), NG/사분위/평균 정합, `renderCharts` HTML(30카드·태그 균형·NaN 없음), SVG 좌표가 viewBox 내, `buildExportSVG` xmlns/치수.
- **CSV 왕복**: 템플릿형 CSV → `parseMeasurementCSV` → `buildQCSheet` → 통계 일치, 빈 템플릿/잘못된 텍스트 폴백.
- **데이터 도구**: `splitCSVLine`(따옴표), 텍스트 나누기, 중복 제거를 mock state로 검증.
- **회귀 방지 교훈**: 코드 구조를 리팩터하면 "슬라이스 기준 문자열"에 의존한 옛 테스트 하네스가 깨진다(예: `genQCData`를 `buildQCSheet`로 분리하니 옛 슬라이스가 DOM 코드까지 포함). → 검증은 **마커 문자열 슬라이스 + DOM 비의존 순수함수** 위주로, 리팩터 후 통합 테스트 재작성.

> 주의: 문법 검증은 `node --check`, 로직 검증은 슬라이스 후 순수 함수 실행. 실제 렌더/래스터화/클립보드/파일 IO는 브라우저에서 수동 확인 필요.

## 12. 성능 노트

- **DOM 그리드**: 64×300 ≈ 19,200 셀을 실제 `<table>`로 생성. 초기 로드 시 이게 가장 큰 비용(레이아웃/페인트). 매우 커지면(수천 행) **가상 스크롤**이 필요 — 현재 미구현.
- **적용한 최적화**:
  - 산점도 점: 수백 개 `<circle>` 대신 정상 점은 **단일 `<path>`**(둥근 캡 점, `M x y h0.01`)로 그려 DOM 요소를 대폭 감소. NG 점만 개별 `<circle>`(+툴팁).
  - 차트 전환/데이터 로드의 인위적 로딩 지연을 `320ms`로 단축(체감 개선).
  - `recalc`는 전체가 아닌 "채운 칸"만 순회.
- **주의**: 선택 테두리 위치는 `td.offsetLeft/Top`(DOM 측정) 기반. 그래서 그리드 행에 `content-visibility:auto`를 쓰면 오프스크린 셀 오프셋이 틀어질 수 있어 **적용하지 않음**. 가상 스크롤 도입 시 선택 좌표를 `colW/rowH` 누적 계산으로 바꿔야 안전.

## 13. 알려진 한계 / 다음 후보

- **크기 한도**: 포인트 ≤ 60(`MAX_POINTS`), 시료 ≤ 287(`ROWS-13`). 초과분은 파싱/빌드에서 잘림(`truncated` 플래그로 토스트 안내). 더 키우려면 COLS/ROWS 상향(+ 가상 스크롤 권장).
- **xlsx**: 신형 `.xlsx`만(ZIP+deflate). 구형 `.xls`(바이너리 BIFF)와 암호화 통합문서는 미지원. `DecompressionStream` 미지원 브라우저에선 에러 토스트.
- **양식 유연화 보류**: 현재는 템플릿 양식(포인트=열, 라벨행 고정) 전제. 포인트를 행으로 두거나 머리글 자동 탐지 등은 다음 과제.
- 셀 텍스트 오버플로 미구현, 병합/필터/피벗/조건부서식/행·열삽입/차트삽입은 더미, 다크모드 없음, 저장(영속) 없음.
- 다음 후보: 양식 유연화, 가상 스크롤, 위저드형 "차트 생성 마법사", 조건부 서식/행·열 삽입.
