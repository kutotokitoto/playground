# 투어 카드 마크다운 변환 스킬 (마법사 마법사)

## 무엇을 하는가

사용자가 **줄글로 늘어놓은 사용법**을 받아서, 마법사 마법사 도구에 그대로 붙여넣을 수 있는
**투어 카드 마크다운**으로 변환한다. 한 카드 = 한 행동. 여러 카드는 `---` 줄로 나눠 출력하고,
사용자는 페이지를 그 수만큼 만들어 블록을 하나씩 각 페이지의 **[마크다운 붙여넣기]** 필드에 넣는다.

핵심 원리(원본 설계): **헤딩 레벨 = 위계.** `#` 제목 → `##` 본문 → `###` 팁.
색·모양은 `preset`(팔레트)과 `style`(카드 형태)로만 바뀌고, 내용 구조는 그대로다.

> **근거 문서 (design specs).** 이 스킬은 사용자가 작성한 두 스펙을 앱의 현재 구현 기준으로
> 통합·최신화한 것이다. 더 깊은 근거가 필요하면 참고:
> - `투어 마크다운 규칙.md` — 문법(헤딩 위계·메타·버튼·인라인)의 원본 규칙
> - `말풍선 렌더링 스펙.md` — 시각 렌더링(type × style × preset 토큰, 부품 B1~B11, 배치 규칙)
>
> **충돌 시 우선순위: 앱의 실제 동작 > 이 스킬 > 원본 스펙.**
> (앱은 스펙 이후로 확장됨 — 스타일 10종, 팔레트 15종, 형광펜 5색, `###` 팁의 손글씨 렌더 등.
> 원본 스펙의 `glass` 스타일·6종 프리셋 표 등 일부는 구버전이므로 아래 레퍼런스를 따른다.)

---

## 변환 절차 (이 순서대로)

1. **행동 단위로 쪼갠다.** 줄글을 사용자가 실제로 하는 동작 순서로 나눠 **3~5장**(필요하면 1~7장)의 카드로.
   한 카드는 한 가지 행동만 담는다. "그리고/그다음"이 나오면 대개 카드 경계다.
2. **테마(스타일)를 하나만 정해 끝까지 유지한다.** 주제에 가장 어울리는 `style:` 하나를 10종 중에서 고르고,
   **모든 카드에 똑같은 `style:` 을 쓴다.** 카드마다 스타일을 바꾸면 말풍선 모양이 왔다갔다 해서 어지럽다.
   변주가 필요하면 색은 `preset`, 기능 구분은 `type`으로만 한다(모양은 고정). 감이 안 오면 `classic`이 무난하다.
3. **첫 카드 = 환영 카드.** (정한 테마 그대로) 제목은 반가운 한 줄, 본문은 전체 흐름 1문장 요약, 버튼 `[시작](next)`.
4. **중간 카드 = 각 단계.** `#` 제목은 **명령형 한 줄**("~해 보세요"), `##` 본문은 1~2문장.
   - 클릭할 **버튼·메뉴·탭 이름**은 `**강조**`
   - 키보드 단축키는 백틱: `` `Ctrl S` ``
   - 그 카드에서 꼭 기억할 핵심 문구 **하나**만 `==형광펜==` (카드당 최대 1곳)
   - 부가 설명·예외·요령은 `### 팁:` 또는 `### 주의:` 로 분리 (카드당 0~1개)
   - 버튼은 하나만: `[다음](next)`
5. **마지막 카드 = 완료 카드.** `type: complete`, 버튼 `[시작하기](done)`. `style:` 은 정한 테마 그대로 둔다.
6. **경고/위험은 `type`으로만 뺀다.** 되돌릴 수 없는 행동은 `type: confirm`(+`danger:` 버튼),
   단순 주의 공지는 `type: banner`. **이때도 `style:` 은 바꾸지 않는다** — 타입만 바꾼다.
7. **메타 한 줄 넣기.** 진행형 투어면 모든 카드 첫 줄에 `step: auto` (페이지 수와 자동 연동).
   색 팔레트도 `preset:` 하나를 **모든 카드에 동일하게**(또는 비워 두고 도구의 팔레트 스와치로) — 색도 왔다갔다 하지 않게.
8. **출력은 마크다운만.** 설명 문장 없이, 카드들을 `---` 로 구분한 하나의 코드블록으로.

정적 이미지라는 점을 기억하라: **`건너뛰기`·`닫기(×)` 같은 동작 버튼·부가 액션은 넣지 않는다.**
버튼은 방향을 가리키는 장식이므로 카드마다 하나면 충분하다.

## 문법 레퍼런스 (자립형 — 이것만으로 충분)

### 카드 한 장의 구조 (위에서 아래로)

```
step: 1/3          ← 메타 줄들 (첫 헤딩 위, 전부 생략 가능)
style: classic
preset: teal

# 제목             ← 카드당 1개  (공백 없이 #제목 도 인식)
## 본문 문단        ← 여러 개면 문단으로 쌓임
### 팁: 부가 설명   ← 손글씨 팁 인셋

[다음](next)       ← 버튼 줄 (맨 마지막 줄)
```

### 내용 3층위 + 인라인

| 문법 | 역할 |
|---|---|
| `# 텍스트` | 카드 제목 (카드당 1개) |
| `## 텍스트` | 본문 문단 (여러 개 가능) |
| `### 텍스트` | TIP 인셋 — 연한 배경 + `TIP` 라벨, **나눔손글씨 펜**으로 렌더 |
| `### 주의: 텍스트` | 라벨을 `TIP` 대신 `주의`로 (임의 라벨: `### 메모:` `### 참고:` `### 팁:` 등) |
| `**텍스트**` | 본문 강조 (진하게) — 클릭할 UI 요소 이름에 |
| `` `Ctrl N` `` | 키보드 단축키 칩 |
| `==텍스트==` | 형광펜 (기본 노랑). 한 줄 안 여러 단어면 이어진 밴드 |
| `==민트:텍스트==` | 형광펜 색 지정 — `노랑` `민트` `분홍` `파랑` `주황` |

- 헤딩 바로 아래 일반 줄은 위 헤딩에 이어 붙는다(줄바꿈 병합).
- `##` 한 줄만 쓰면 자동으로 툴팁처럼 작아진다.

### 메타 줄 (`키: 값`, 첫 헤딩보다 위)

| 키 | 값 | 설명 |
|---|---|---|
| `step` | `2/5` · `auto` · `none` | 진행도 라벨 + 도트. `auto`면 현재/전체 쪽수 자동 연동 |
| `step-label` | 예: `STEP 2/5` | 진행도 표기 커스텀 |
| `style` | 아래 10종 | 카드 형태 |
| `preset` | 아래 15종 | 색 팔레트 (구조 불변, 색만) |
| `type` | `tour-step`(기본) `tooltip` `confirm` `toast` `banner` `chip` `hotspot` `complete` | 카드 형태 변형 |
| `close` | `true` | 우상단 × 표시. **기본은 숨김** — 정지 이미지라 대개 불필요 |
| `media` | `placeholder` | 카드 상단 미디어(스크린샷 자리) 영역 |

### 버튼 줄 (맨 마지막 줄)

- `[라벨](액션)` 을 나열 — **마지막 버튼이 강조색**(프라이머리), 나머지는 고스트.
- 액션 키워드: `next` `prev` `skip` `done` `dismiss` `undo` (그림용, 실제 동작 없음)
- `[삭제](danger:del)` → 빨간 파괴 버튼 · 라벨이 `→`로 끝나면 박스 없는 텍스트 링크
- 버튼 줄이 없으면 푸터 자체가 생략된다.

## 스타일 고르기 (`style:`) — 10종

| style | 생김새 | 언제 |
|---|---|---|
| `classic` | 흰 카드 + 헤어라인 + 부드러운 그림자 | **기본값. 대부분의 단계 카드** |
| `accent-header` | 상단에 강조색 헤더 바(진행도·제목 포함) | **환영/시작 카드**, 섹션 첫 카드 |
| `dark` | 어두운 표면 + 밝은 글자 | 어두운 배경 위, 강한 대비가 필요할 때 |
| `soft-tint` | 강조색 옅은 배경 + 플랫 | 부드러운 참고·보조 설명 |
| `mist` | 강조색이 좌상단→우하단 은은히 옅어지는 그라데이션 | 감성적·환영 톤, 배경이 살짝 비치는 느낌 |
| `outline` | 두꺼운 윤곽선 + 점선 팁, 그림자 없음 | 담백·미니멀, 라인 아트 톤 |
| `pop` | 굵은 보더 + 오프셋 그림자 + `STEP n/m` 필 | **강한 주목**, 축하·신규 강조 |
| `vivid` | 솔리드 강조 표면 + 흰 글자 + 뒤 유령 카드 + **오른쪽 큰 각진 화살표** | 강렬한 시작/CTA, 특정 위치를 크게 가리킬 때 |
| `divider` | 본문과 팁 사이 **점선 구분선** + 부드러운 꼬리 | 정보가 많아 본문·팁을 또렷이 나눠야 할 때 |
| `window` | 상단 **브라우저 창 헤더 바 + 신호등 점 3개**(팔레트색) | 브라우저/환경 권장, 시스템·설치 안내 |

> **⚠️ 한 투어 = 한 스타일.** 시작할 때 위 표에서 주제에 맞는 것 **하나**를 고르고, 그 투어의 **모든 카드**에
> 같은 `style:` 을 쓴다. 카드마다 바꾸면 말풍선이 왔다갔다 해 어지럽다. 색 변주는 `preset`, 기능 구분은
> `type`(완료·확인 등)으로 하되 모양(`style`)은 고정. 감이 안 오면 `classic`.

## 팔레트 고르기 (`preset:`) — 15종, 의미색으로

`light`(인디고, 기본) · `blue` `sky` `cyan` `teal` `emerald` `lime` · `yellow` `amber` ·
`rose` `fuchsia` · `violet` · `slate` `mono` · `dark`(다크 테마 전체)

- **신뢰·정보·안정** → `light` `blue` `sky` `cyan`
- **성공·완료·긍정** → `teal` `emerald` `lime`
- **주의·순서·시간** → `yellow` `amber`
- **위험·삭제·강조 경고** → `rose` `fuchsia`
- **프리미엄·창의** → `violet`
- **중립·문서·미니멀** → `slate` `mono`
- **다크 테마** → `dark`

팔레트는 보통 **한 투어 전체에 하나**로 통일한다. 특정 카드만 다른 색이 필요한 게 아니라면
개별 카드에 `preset`을 남발하지 말고, 첫 카드에만 지정하거나 아예 비워 두고 도구 스와치로 고른다.

## 형광펜 색 의미 (`==색:텍스트==`)

| 색 | 쓰임 |
|---|---|
| `노랑`(기본) | 가장 중요한 핵심 한 곳 |
| `민트` | 완료·성공·긍정("됐어요") |
| `분홍` | 주의 환기·삭제 관련 |
| `파랑` | 정보·링크성 |
| `주황` | 경고·시간/순서 |

**카드당 형광펜은 최대 1곳.** 남발하면 강조가 사라진다.

## 타입 고르기 (`type:`)

- `tour-step`(기본) — 일반 단계 카드
- `tooltip` — `##` 한 줄이면 다크 미니 툴팁, `#`+`##`이면 작은 카드
- `confirm` — `#` 질문 + `##` 설명 + 버튼 2개(마지막에 `danger:` 권장). 되돌릴 수 없는 행동
- `toast` — `##` 한 줄 + 선택적 `[실행 취소](undo)`. 완료 알림
- `banner` — 경고색 배너, `#` 헤드라인 + `##` 상세. 주의 공지
- `chip` — `## NEW: 내용` 형식의 알약형 힌트
- `hotspot` — `##` 한 줄 = 펄스 비컨 + 미니 라벨
- `complete` — 완료 카드(체크 아이콘). 투어의 마지막

## 반드시 지킬 것 / 하지 말 것

**DO**
- 제목은 짧은 명령형 한 줄, 카드당 하나.
- 클릭 대상 이름은 `**강조**`, 단축키는 백틱, 핵심 한 곳만 `==형광펜==`.
- 부가 설명은 `### 팁:`/`### 주의:` 로 분리.
- 카드마다 버튼 하나. 마지막 카드는 `type: complete` + `[시작하기](done)`.
- 여러 카드는 `---` 로 구분, 출력은 코드블록 하나.

**DON'T**
- **카드마다 `style` 을 바꾸지 않는다 — 한 투어는 한 테마.** (색은 `preset`, 기능은 `type`으로만 변주)
- `건너뛰기`·`닫기` 등 동작 버튼/`close: true` 를 습관적으로 넣지 않는다(정적 이미지).
- 한 카드에 형광펜을 두 곳 이상 쓰지 않는다.
- 본문을 3문장 이상으로 늘리지 않는다 — 길면 카드를 쪼갠다.
- 마크다운 앞뒤에 설명 문장을 붙이지 않는다(코드블록만).
- 강조원(스포트라이트)·회전·이미지·팔레트 스와치는 마크다운이 아니라 **속성 패널/핸들**에서 조절 — 마크다운으로 만들려 하지 않는다.

## 변환 예시

### 예시 A — 한 테마로 3단계 (환영 → 단계 → 완료)

**입력(줄글):** "먼저 데이터 탭에서 파일을 올리고요, 컨트롤 브이로 붙여넣어도 돼요.
그다음 정규분포 탭에서 그래프를 보면 되는데 빨간 점이 불량이에요. 다 보면 저장 버튼으로 PNG를 받으세요."

**출력:** (차분한 안내라 테마 `soft-tint` 하나를 골라 **모든 카드**에 동일하게 — 마지막만 `type`으로 완료 처리)
```
step: auto
style: soft-tint

# 환영해요!
## 3단계로 데이터 업로드부터 저장까지 안내해 드릴게요.

[시작](next)
---
step: auto
style: soft-tint

# 데이터를 올려 보세요
## **[데이터]** 탭에서 파일을 업로드하세요.
### 팁: `Ctrl V` 붙여넣기로도 올릴 수 있어요

[다음](next)
---
step: auto
style: soft-tint

# 그래프를 확인하세요
## **정규분포** 탭에서 분포를 확인하세요. ==분홍:빨간 점==이 불량이에요.

[다음](next)
---
type: complete
style: soft-tint

# 끝났어요!
## **저장** 버튼으로 PNG를 내려받으세요.

[시작하기](done)
```

### 예시 B — 위험 단계는 `style` 고정 + `type`만 바꾸기
